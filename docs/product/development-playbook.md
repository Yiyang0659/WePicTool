# WePicTool 开发行动手册

这份手册用于后续给开发者或 AI 编码助手下任务。核心原则是一次只推进一个阶段，不把后续能力提前混进当前阶段。

## 固定任务模板

```text
1. 【背景】当前处于哪个阶段，已有能力是什么
2. 【本次任务】只实现本阶段目标
3. 【技术栈】原生微信小程序 + CloudBase
4. 【约束】引用 PRD 开发红线
5. 【验收标准】列出真机要验证的动作
```

## 当前阶段说明

当前项目处于阶段二和阶段三之间：

- 阶段一工程骨架已完成。
- 阶段二 AI 分类已接入（DashScope `qwen-vl-plus`），用户可手动修改分类。
- 阶段三的抠图 API 评估要求放在 `docs/ai-workflows/matting-api-evaluation.md`。

## 阶段一任务口径

```text
我在做一个微信小程序，当前任务是跑通穿搭参考图的最小技术闭环。

需求：
1. 首页有主按钮“做穿搭参考图”
2. 调用 wx.chooseMedia，支持选 1-9 张图片
3. 上传前本地压缩，最长边不超过 1600px
4. 将图片上传到 CloudBase 云存储
5. 调用 processOutfit 云函数创建处理任务
6. 云函数先不做真实 AI，只把原图按 mock 分类返回
7. 前端展示分组结果页
8. 结果页提供“保存单张”和“保存当前组”

约束：
- 不做登录
- 不做真实抠图
- 不做复杂 UI
- 保存相册必须处理授权被拒
```

## 阶段二任务口径

```text
在已有最小闭环基础上，加入穿搭部件识别能力。

需求：
1. 云函数拿到图片后调用选定的分类 API/模型
2. 返回 type 和 confidence
3. confidence >= 0.8 直接分组
4. confidence < 0.8 交给用户确认
5. 结果页允许用户修改分类

约束：
- 完整人物试穿图默认 daily
- 头像、包、配饰等非 MVP 素材不进入穿搭主链路
- 分类结果必须可以被用户手动覆盖
```

## 阶段三任务口径

```text
实现穿搭白底卡片生成链路。

需求：
1. 只对 tops/bottoms/shoes 做白底卡片生成
2. 调用选定抠图 API 获取透明主体图
3. 后端合成 1:1、4:5、3:4 三种比例
4. 主体最长边占画布 78%-86%
5. 白色/浅色衣物自动加轻阴影、细描边或浅灰底
6. 单张失败不影响其他图片展示

约束：
- 后端合成，不用前端 Canvas 处理大图
- MVP 不加水印
```

## 每次开发前检查

1. 当前任务是否只属于一个阶段。
2. 是否引用了 `docs/product/PRD.md` 的开发红线。
3. 是否有真机验收动作。
4. 是否避免把日常合集、表情包、心情状态图提前放入主入口。
5. 是否保留每组不足 3 张时的降级提醒。
