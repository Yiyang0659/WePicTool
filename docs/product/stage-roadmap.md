# WePicTool 阶段路线

**当前口径：** 先完成阶段一真机验收，再接阶段二图片部件识别和阶段三白底卡片生成。

| 阶段 | 目标 | 当前状态 | 验收重点 |
| --- | --- | --- | --- |
| 阶段零：开发前准备 | 选定抠图 API、确认分类方案、创建 CloudBase 环境 | 业务确认中 | CloudBase 可上传、云函数可调用、真机可保存相册 |
| 阶段一：最小闭环 | 选图、压缩、上传、mock 分组、结果页、保存 | 工程完成，本地预览和云开发链路均可用 | 选 1-9 张图可进入结果页，单张保存和按组保存可用 |
| 阶段二：图片部件识别 | 自动识别 tops/bottoms/shoes/other_product/daily/unsupported/uncertain | 已完成（AI 分类已接入，用户可手动修改分类） | 高置信度自动分组，低置信度可人工确认 |
| 阶段三：白底穿搭卡片 | 对上衣、下装、鞋子做抠图、白底、比例和主体排布 | 抠图已接入（DashScope qwen-image-2.0），白底卡片合成待实现 | 白底结果可保存，浅色衣物可辨认，单张失败不阻断 |
| 阶段四：结果页增强 | 微信叠图效果预览、单组滑动、保存引导增强 | 基础骨架已有 | 每组不足 3 张时清晰降级提示 |
| 阶段五：轻编辑与兜底 | 改分类、删除、重做、排序、切比例、埋点 | 未开始 | 用户不用重新开始即可修正错误 |
