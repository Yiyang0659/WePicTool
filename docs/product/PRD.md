**版本：** v1.3-stage  
**日期：** 2026-07-07  
**状态：** 阶段二 AI 分类已接入，阶段三待接入抠图与白底卡片生成。

## 产品定位

WePicTool 是微信里的穿搭参考图生成工具。用户选择上衣、下装、鞋子等图片，小程序自动整理为分组素材，方便保存后回微信按组发送，让朋友滑动对比并给出搭配建议。

这个产品不是轻量版修图工具，也不是泛图片工具箱。MVP 的核心任务是缩短这条路径：

```text
选图 -> 自动整理成上衣/下装/鞋子组 -> 按组保存 -> 回微信发送给朋友参考
```

## MVP 主线

MVP 只做“上衣 / 下装 / 鞋子穿搭求建议素材包”主链路。通用商品白底能力作为底座保留，头像、包、配饰、日常合集、表情包、心情状态图、付费能力不进入首版主入口。

**关于历史记录：** 主入口不做云端同步的历史记录体系；允许保留设备本地的轻量记录列表（`pages/record`），仅用于快捷查看和再次生成，不上云、不关联账号、不同步。

## 目标用户

| 用户 | 场景 | 主要诉求 |
| --- | --- | --- |
| 穿搭求建议型用户 | 买衣服、出门前搭配、网购下单前 | 快速整理候选项，让朋友容易比较 |
| 闺蜜挑选型用户 | 试了几套衣服，想问朋友哪套更好 | 朋友能回复“上衣 2 + 下装 1” |
| 个人卖货 / 小红书起步用户 | 拍了衣服、鞋、小物件 | 得到干净、统一、可保存的商品图底座 |

## 当前工程状态

- 首页支持选择 1-9 张图片。
- 未配置 CloudBase 时支持本地预览模式。
- 已有 `processOutfit` 云函数，阶段一 mock 分组和阶段二 AI 分类均可用。
- 云函数接入 DashScope `qwen-vl-plus` 多模态模型做真实图片分类。
- 结果页支持上衣组、下装组、鞋子组、未处理素材区展示。
- 结果页支持单张保存和按组保存。
- 结果页支持「改分类」，低置信度图片显示「待确认」角标。
- 点击图片可预览大图。
- 每组不足 3 张时会降级提醒，避免误导用户认为一定能形成微信叠图效果。

## 开发红线

1. 不做复杂画布编辑器。
2. 不承诺多图一键直发聊天。
3. 不把日常合集、表情包、心情状态图放进 MVP 主入口。
4. 不让单张失败阻断整批任务。
5. 不提前引入账号、云端同步历史记录、付费体系。
   - 设备本地轻量记录（`pages/record`）仅用于快捷查看和再次生成，不上云、不关联账号、不同步。
6. 不把商品分类做得过细，阶段二只围绕穿搭主链路分类。
7. 用户上传图片仅用于本次处理，正式版本需补齐 24-72 小时清理说明。

## 不在当前阶段

- 真实 AI 分类调用。
- 真实抠图 API 调用。
- Sharp/Pillow 白底合成。
- 复杂画布编辑。
- 历史记录、账号体系、会员付费。
  - 注：设备本地轻量记录已作为体验增强加入，不属于云端同步历史记录。
- 头像、包、配饰、日常合集、表情包、心情状态图主入口。
