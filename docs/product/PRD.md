**版本：** v2.0
**日期：** 2026-07-13
**状态：** 阶段三进行中（白底卡片合成待实现）

## 1. 产品定位

### 1.1 一句话定位

> 微信里最快的穿搭参考图生成工具：用户选好上衣、下装、鞋子等图片，系统自动抠图、白底、分类、统一成分组卡片，方便发给朋友滑动帮忙选搭配。

### 1.2 产品形态

微信小程序，无需登录，打开即用。

### 1.3 解决的核心问题

用户想让朋友帮忙参考穿搭时，现有链路太长：

```text
找图 / 截图 / 拍照
  → 打开第三方修图工具
  → 建白底画布
  → 手动导入衣服
  → 抠图、擦边、对齐、导出
  → 回微信逐张发送
  → 朋友再理解怎么对比
```

产品要把它压缩成：

```text
选图
  → 自动整理成上衣组 / 下装组 / 鞋子组
  → 按组保存
  → 回微信发送给朋友滑动对比
```

### 1.4 第一性原理

用户不是为了"修图"而来，而是为了更快完成一个社交任务：

> 我想让朋友帮我判断哪套穿搭更合适。

因此 MVP 的核心不是做一个轻量版醒图，也不是泛图片处理工具，而是缩短"准备素材 → 发给朋友 → 获得反馈"的路径。

### 1.5 阶段边界

- **MVP 阶段：** 先做好"上衣 / 下装 / 鞋子穿搭求建议素材包"一条主链路，并保留"通用商品白底图"作为底座能力。
- **MVP 不主做：** 头像抠图、包 / 配饰、日常照片合集、表情包、心情状态图、复杂模板、历史记录云端同步、付费体系。
- **后续阶段：** 在出图稳定后扩展头像、包 / 配饰、日常合集、微信合并图增强、表情包、心情状态分享等互动素材。

## 2. 目标用户

### 2.1 用户画像 A：穿搭求建议型

- **场景：** 买衣服、出门前搭配、网购下单前，想让朋友帮忙看哪件上衣配哪条下装。
- **痛点：** 素材散、背景乱、用醒图处理步骤多，发到微信后朋友不容易对比。
- **诉求：** 自动抠图、白底、分组、统一大小，能直接发给朋友滑动看。

### 2.2 用户画像 B：闺蜜挑选型

- **场景：** 试了几套衣服，想发给闺蜜问哪个好看。
- **痛点：** 原图背景乱、角度不一，发出去显得随便。
- **诉求：** 快速整理成可比较的选项，朋友能直接说"上衣 2 + 裤子 4"。

### 2.3 用户画像 C：个人卖货 / 小红书起步型

- **场景：** 拍了衣服、鞋包、饰品、小物件，想发朋友圈、小红书、微信群。
- **痛点：** 不会 PS，不想下载大 App，但希望图片看起来像正经商品图。
- **诉求：** 干净白底、主体居中、可批量保存。

### 2.4 后续扩展用户画像 D：微信高频分享型

- **场景：** 旅游、聚会、日常状态分享。
- **痛点：** 原图大小不一、没整理感。
- **诉求：** 日常照片自动排版成合集图。

## 3. MVP 定义

### 3.1 MVP 产品定义

> 用户上传 1-9 张穿搭相关图片，系统自动识别上衣、下装、鞋子等类别，生成统一白底分组卡片，用户按组保存后回微信发送给朋友滑动参考。

### 3.2 MVP 只做这些

| 模块 | 功能 |
| --- | --- |
| 首页 | 主入口：做穿搭参考图 |
| 上传 | 相册选择 1-9 张，上传前压缩 |
| 识别 | 上衣 / 下装 / 鞋子 / 其他 / 日常 / 非 MVP 支持 / 不确定；MVP 仅处理前三类 |
| 处理 | 抠图、白底、比例选择、主体居中、组合视觉锚点 |
| 预览 | 分组卡片 + 微信叠图效果模拟 + 分类后发送效果校验 |
| 轻编辑 | 改分类、删除、重做、调整顺序、切换比例 |
| 导出 | 单张保存、按组保存、保存全部、一键合并发送、发送引导 |

### 3.3 MVP 不做这些

- 账号 / 登录 / 注册系统
- 历史记录云端同步、云端长期保存
- 付费墙 / 会员体系
- 表情包制作主入口
- 心情状态图主入口
- 日常照片合集主入口
- 复杂编辑器
- 素材市场 / 模板商城
- 社区 / 分享广场
- App
- 小红书 / 闲鱼等平台专用尺寸模板

> 注：设备本地轻量记录（`pages/record`）仅用于快捷查看和再次生成，不上云、不关联账号、不同步，不属于云端同步历史记录。

### 3.4 首版成功标准

> 用户第一次打开，不用学习，能看到清晰进度，并在合理等待后得到一组可以发给朋友参考穿搭的分组白底图。

具体 Demo 成功标准：

- 3 张图 20 秒内有结果。
- 9 张图 60 秒内有结果，并且进度明确。
- 上衣/下装/鞋子分类可人工修正。
- 白底结果至少 70% 值得保存。
- 白色/浅色衣物在白底上可辨认。
- 用户可以顺利按组保存。
- 用户理解如何回微信发送。
- 分类后每组少于 3 张时能看到降级提醒；总数够但每组不足时不误导用户形成叠图。

## 4. 核心用户流程

```text
进入小程序
  → 点击"做穿搭参考图"
  → 选择 1-9 张图片
  → 前端压缩 + 上传
  → AI 自动识别图片类型和穿搭部件
    → 上衣 / 下装 / 鞋子 / 其他商品 / 日常图 / 非 MVP 支持 / 不确定
  → 对上衣 / 下装 / 鞋子自动抠图 + 白底合成 + 用户选择输出比例
  → 结果预览
    → 按组展示：上衣组 / 下装组 / 鞋子组 / 未处理素材区
    → 模拟微信叠图滑动效果
    → 分类后校验每组是否满足微信叠图发送条件
  → 轻编辑
    → 改分类 / 单张重做 / 删除 / 调整顺序 / 切换比例
  → 按组保存到相册
  → 引导回微信按组发送
```

### 4.1 关键体验原则

1. 用户不需要先理解"抠图""白底""分类"等工具概念。
2. 系统默认帮用户分组，错了允许用户快速改。
3. 结果页必须让用户一眼看懂"发到微信后会怎么滑动对比"。
4. 不承诺一键把多张图直接发进聊天，MVP 先做"按组保存 + 发送引导"。
5. 上传 1-9 张是 MVP 技术上限，不代表一定能形成微信叠图；微信合并展示按分类后的每个发送组判断。
6. 微信合并展示依赖每组 3 张及以上素材；分类后不足 3 张时必须降级提醒用户"可保存单图，但可能无法形成合并滑动效果"。
7. 如果总图数 >= 3，但分类后每组都不足 3 张，必须提示"当前更适合普通发送；想要叠图效果，建议每组补到 3 张以上"。

## 6. 非功能需求

### 6.1 性能要求

| 场景 | 要求 |
| --- | --- |
| 1 张图端到端处理 | <= 10 秒，暂定 |
| 3 张图端到端处理 | <= 20 秒，暂定 |
| 9 张图端到端处理 | <= 60 秒，全程进度可见，暂定 |
| 前端压缩耗时 | <= 2 秒/张 |

> 以上耗时阈值先作为体验目标，后续根据分类模型、抠图 API、云函数合成速度评测结果修订。

### 6.2 可靠性要求

- 单张失败不能导致整批任务中断。
- 抠图 API 超时超过 15 秒需明确提示。
- 批量保存失败时显示失败数量，并允许逐张重试。
- 分类错误必须允许用户手动修正。

### 6.3 隐私与安全

- 用户上传图片仅用于本次处理。
- 原图和结果图均临时存储，24-72 小时自动清理。
- 进入小程序时用简短文案说明图片处理和清理策略。
- MVP 不做账号体系，不保存用户身份信息。

### 6.4 兼容性

- 支持 iOS / Android 主流机型。
- 低版本微信对保存/分享能力降级提示，不报错。
- 首版支持 JPG / PNG，HEIC 后端转码兜底。

## 7. 开发红线

### 7.1 产品规则

1. MVP 主入口必须围绕"穿搭参考图"，不要做成泛图片工具箱。
2. 默认先选图，再由系统自动识别和分组。
3. 识别结果必须可以被用户手动修改。
4. 试穿全身图默认不抠整个人，避免生成怪异白底人像。
5. 穿搭卡片支持 1:1 / 4:5 / 3:4，用户可在预览页选择。
6. 批量保存必须可中断、可重试、可看到失败数量。
7. 结果图 MVP 阶段不加水印。
8. 用户上传图片仅用于本次处理，24-72 小时自动删除。
9. MVP 只处理上衣、下装、鞋子三组；其他商品、头像、包、配饰、日常图、未确认素材后续再做。
10. 每个需要微信合并滑动展示的组至少需要 3 张素材，必须在分类完成后按组校验，不足 3 张必须降级提醒。
11. 如果用户上传总数 >= 3 但分类后每组不足 3 张，不能暗示可以形成视频里的叠图效果，只能提示普通发送或补图。
12. 输出比例不在文档中拍死默认值，首版提供 1:1 / 4:5 / 3:4 让用户预览选择，后续由真机测试决定推荐值。
13. 白色或浅色衣物在白底上的可见性增强是 P0，不得后置。

### 7.2 开发红线

1. 不做复杂画布编辑器。
2. 不把"分享"承诺成多图一键直发聊天，除非真机验证微信能力可行。
3. 不把日常合集、表情包、心情状态图放进 MVP 主入口。
4. 不在前端处理大图合成（白底卡片合成除外），稳定性不可控。
5. 不把商品分类做得太细，错分比不分更伤用户信任。
6. 不让单张失败阻断整批任务。
7. 不过早引入账号、云端同步历史记录、付费体系。
8. 不忽视用户上传身体、穿搭、商品图片的隐私说明。

## 9. 成功指标

| 指标 | 定义 | MVP 目标阈值 |
| --- | --- | --- |
| 穿搭素材包完成率 | 用户选图后成功生成分组卡片的任务比例 | >= 85% |
| **按组保存率** | 用户生成后保存至少一组结果图的比例 | **>= 50%** |
| **分享意愿率** | 用户点击"去微信发送 / 分享提示"的比例 | **>= 25%** |
| 商品白底可用率 | 内测中白底结果值得保存的比例 | >= 70% |
| 单次处理耗时 | 3 张 <=20s，9 张 <=60s；30 秒内必须有明确进度反馈 | 暂定，模型速度评测后修订 |
| 次日复用率 | 第二天再次打开的用户比例 | 有明显信号 |

> MVP 不以功能数量判断成败，只看用户是否愿意保存、发给朋友、下次还回来。
