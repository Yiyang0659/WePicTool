## 当前产品判断

现在项目已经从“不能跑”进入“可用的技术闭环”，并且 UI 已升级为更贴近微信真实发送体验的预览形态：

- 小程序采用底部三 Tab（首页 / 记录 / 我的）。
- 结果页改为微信聊天窗口预览风格，上衣 / 下装 / 鞋子以“合并发图”气泡卡片展示，点击可展开横向滚动查看全部图片。
- 用户可以提前看到“发给朋友后”的真实效果，增强分享动机。
- 图片只是被分组和保存，还没有真正变干净。
- 分组仍是 mock，不是 AI 判断。
- 结果图还没有抠图、白底、主体居中、统一比例。
- 用户能完成流程，但还没有得到“值得发给朋友参考”的最终穿搭素材。

## 产品迭代优先级

### P0：分类可信度

- 从 mock 分组改为 AI 分类。
- 标签固定为 `tops`、`bottoms`、`shoes`、`other_product`、`daily`、`unsupported`、`uncertain`。
- `confidence >= 0.8` 自动分组。
- `confidence < 0.8` 让用户确认。
- 完整人物试穿图默认 `daily`，不进入白底人像链路。

### P0：出图质量

- 对 `tops`、`bottoms`、`shoes` 做抠图。
- 生成白底图。
- 主体居中，大小统一。
- 白色/浅色衣物需要轻阴影、描边或浅灰底兜底。
- 单张失败不阻断整批。

### P1：结果页轻编辑

- 改分类（已可用）。
- 删除单张。
- 单张重做。
- 保存本组 / 保存全部 / 按分组保存。
- 编号更明显，方便朋友反馈“上衣 1 + 下装 2”。
- 微信聊天预览风格的合并发图展示与展开交互。

### P1：微信发送引导

- 保存后提示“先发上衣组，再发下装组，最后发鞋子组”。
- 结果页直接模拟微信聊天窗口效果，降低用户想象成本。
- 每组不足 3 张时提示普通发送。
- 总图数够但每组都不足 3 张时，不暗示能形成微信叠图。

### P2：数据指标

后续至少记录：

- `task_created`
- `classification_completed`
- `category_changed`
- `task_completed`
- `group_saved`
- `result_saved`
- `share_guide_clicked`
- `retry_triggered`
