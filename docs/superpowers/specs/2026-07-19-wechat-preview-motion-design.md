# 微信预览页动效与比例兼容设计

**日期：** 2026-07-19

## 目标

将 `miniprogram/pages/preview/` 调整为白底的真实微信聊天预演：会话名为「分享给好友」，时间显示为 12:00；在一屏内可看到上衣、下装、鞋子三组折叠卡；滑动、补位与展开/收起的手感接近用户提供的真实微信录屏。

## 已确认的体验口径

- 采用用户提供的真实微信聊天场景作为结构、间距与动效的视觉基准；保留三组纵向消息流、右侧头像、底部输入栏和左侧展开胶囊。
- 聊天背景、导航和输入栏统一为白色，不保留提示条、亮暗主题切换或模拟聊天文案；导航标题改为「分享给好友」。
- 状态栏和消息时间固定展示为 `12:00` / `中午12:00`。
- 静止态的两张后卡分别向左、向右露出一角；展开胶囊向牌堆靠近，但不覆盖图片。
- 用户可水平滑动顶层卡片；未达到阈值时自然回弹，达到距离或速度阈值时按手势方向飞出，后卡连续补位并循环。
- 点击「展开 N」时，第 2 至 N 张卡以微信消息流形式向下展开；点击「收起」时逆序收起。

## 布局与比例兼容

### 稳定交互框

每个分组使用固定的交互舞台：宽度为窗口宽度的 54%，最大视觉高度为 150px（按屏宽等比缩放并设置上限）。三组消息行各预留 164px，卡片视觉尺寸相对上一版约放大三分之一；小屏首屏允许第三组露出后继续向下滚动，保持真实聊天页面的密度。

舞台尺寸不随当前顶层图片变化，因此翻页不会因不同图片比例而导致消息流跳动；后卡也始终在同一个手势命中区域内。

### 图片呈现

每张卡根据已有比例元数据优先使用 `composedRatio`，其次使用任务的 `ratio`，最后使用图片宽高推导。卡片本体在稳定交互框中等比居中，宽高均不超过舞台边界；图片使用 `aspectFit`，不拉伸、不裁切。不同来源的 1:1、4:5、3:4、原图兜底和未知比例都不会撑破或压缩聊天流。

同一组内若卡片比例不同，舞台保持不变，单张卡在其中居中适配；翻页时仅替换内容与视觉比例，消息行高度不变。

## 动效参数

| 项目 | 规格 |
| --- | --- |
| 顶层卡宽舞台 | `54vw`，最大高 150px |
| 消息行高度 | 164px（按视觉密度微调，不随图片比例变化） |
| 后卡露边 | 左侧 `-8px`、右侧 `+13px`，缩放 `.97` / `.94`，旋转 `-.7°` / `+1.1°` |
| 方向锁 | 首次位移超过 8px 后，水平位移大于垂直位移才接管手势 |
| 跟手 | `translateX(dx)` + `rotate(dx * .025°)`，旋转限制为 ±6° |
| 触发翻页 | 位移超过卡宽 20%，或最近 120ms 速度超过 `0.28px/ms` |
| 回弹 | 240ms，`cubic-bezier(.18,.82,.2,1)` |
| 飞出 | 220ms，向手势方向飞出 `1.3 × 卡宽`、旋转 ±18°、渐隐 |
| 补位 | 同一 240ms transform 动画，固定三节点仅轮转 class；飞出节点无过渡落到后层 |
| 展开 | 第 2 至 N 张以 220ms 向下位移 10px + 透明度动画进入，45ms stagger |
| 收起 | 180ms ease-in，30ms 逆序 stagger；收起后隐藏消息行，不销毁图片节点 |

## 图片稳定性与预加载

- 折叠牌堆、展开的首张卡和其余扩展卡在页面首次渲染时均保留对应的图片节点；展开/收起只切换可见性、触摸能力和动画 class，禁止通过 `wx:if` / `wx:else` 重建图片。
- 所有预览图片关闭懒加载，使隐藏节点在用户首次点击前完成加载和解码；图片异常仍沿用现有失败占位逻辑。
- 该策略优先保障点击后的连续画面，代价是预览页会常驻更多已解码图片；页面只展示用户刚生成的一次分组结果，数量受已有分组限制，内存开销在当前范围内可接受。

## 实现边界

- 仅修改预览页及其契约文档、必要的纯函数测试；不改变结果页的保存、分组、eventChannel 传参和长按保存/转发能力。
- 保持固定三节点轮转，禁止为每次滑动重建牌堆节点。
- 手势与卡片动画只使用 `transform` / `opacity`；不得在拖动与补位过程中修改布局尺寸。
- 允许 `theme` 输入继续被接收以兼容旧调用方，但预览页始终渲染确认后的白色主题。

## 验收

1. 375px 宽模拟器和真机中，折叠态同时可见上衣、下装、鞋子三组，底部输入栏仍可见。
2. 同时混用 1:1、4:5、3:4、未知比例和原图兜底时，图片完整、不卡顿、不变形、不裁切；横向滑动仍稳定。
3. 静止时两张后卡从左右露出可见角；展开胶囊与牌堆间距紧凑但不重叠。
4. 慢拖未过阈值平滑回弹；快速短滑或达到距离阈值均能飞出；连续左右滑动无闪烁、无节点错乱、无纵向滚动抢手势。
5. 首次看到的折叠卡在点击展开或收起时不出现白底、空白图或重新解码式跳变；断网后已进入预览页的缓存图片仍保持已绘制内容。
6. 展开、收起、长按保存、长按后的 tap 抑制和大图查看维持现有功能。
7. `npm run check:syntax`、`npm run check:miniprogram` 和 `npm test` 通过；在微信开发者工具和 iOS/Android 真机对照用户提供视频完成手感验收。
